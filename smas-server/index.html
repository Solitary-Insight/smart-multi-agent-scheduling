The Mermaid diagram models SEMESTER_CONFIGURATIONS as a structured JSON payload inside the configurations.value field. That is the cleanest way to show the schema while preserving your real storage design.

What the semester JSON means

Semester window

semesterName, startDate, and endDate define the active term.

Credit policy

maxCredits and minCredits control enrollment limits.

Schedule grid

slotDuration, dayStart, dayEnd, and break_between_classes define the timetable.

Workflow controls

allowConflicts, autoNotify, requireApproval, and enrollmentOpen drive enforcement and alerts.

Capacity rules

maxClassSize limits class population.

Week setup

weekStart stores the preferred first day of the week.

JSON value as stored in configurations.value

          
          "SEMESTER_CONFIGURATIONS": {
            "semesterName": "string",                  // Name of the semester, e.g., "Fall 2026"
            "startDate": "date",                       // Semester start date, e.g., "2026-09-01"
            "endDate": "date",                         // Semester end date, e.g., "2026-12-15"
            "maxCredits": "number",                    // Maximum credits a student can take
            "minCredits": "number",                    // Minimum credits a student must take
            "slotDuration": "number",                  // Duration of each class slot in minutes
            "break_between_classes": "number",         // Break duration between consecutive classes in minutes
            "dayStart": "time",                        // Academic day start time, e.g., "08:00"
            "dayEnd": "time",                          // Academic day end time, e.g., "18:00"
            "allowConflicts": "boolean",               // Whether schedule conflicts are allowed
            "autoNotify": "boolean",                   // Automatic notifications enabled
            "requireApproval": "boolean",              // Enrollment requires approval
            "maxClassSize": "number",                  // Maximum students per class
            "enrollmentOpen": "boolean",               // Whether enrollment is currently open
            "weekStart": "id(week_days)"               // References the primary key in week_days table for first day of the week
          }

This JSON is the live configuration model for the semester settings entry with key SEMESTER_CONFIGURATIONS.

Structural interpretation

  • configurations is the parent storage table.
  • SEMESTER_CONFIGURATIONS is the logical model embedded in its JSON value.
  • This makes the schema flexible without needing one table column per setting.
  • The rest of the schema remains normalized for users, departments, courses, availability, and scheduling.

Core relationships

  • users → students / teachers as role profiles.
  • departments → courses and students.
  • teachers → courses and availability tables.

Join tables

  • teacher_departments
  • student_courses
  • course_prerequisites
  • course_merge_partners
  • break_days, break_departments
